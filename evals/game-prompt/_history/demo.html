Methodology

The prose prompt (source.md) was written first: a 1,415-word specification with exact physics values, visual design requirements, game state descriptions, Web Audio synthesis specs, and technical constraints. The specificity is intentional — a vague prompt would let each AI build differently, making comparison meaningless.

The BOTSPEAK prompt (source-botspeak-v2.md) was produced by compressing source.md using the updated /botspeak skill. Every exact value (hex colors, pixel dimensions, Hz frequencies, frame counts) was preserved verbatim. The v2 version achieves 62% compression — 12% better than the manual iter1 version. Repeated identifiers were aliased (CV = canvas, BD = bird, PP = pipes).

Both games were built by a fresh AI agent given only the prompt — no existing code, no reference to the other build. The comparison table above shows whether the same constraints were extracted from both formats.
