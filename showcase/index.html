BOTSPEAK · Evaluations

How AIs should talk to other AIs.

BOTSPEAK exists to give AIs a way to talk to other AIs in something closer to their native voice. The Flappy Bird and Snake and Pong and Breakout demos on this page are stress tests for the language's fidelity. They are diagnostic instruments. They are not the product.

What this page tries to show is that a fresh model with no prior context, given only a BOTSPEAK-compressed prompt, produces a game that matches the prose-built game on every numeric constant and every behavioral rule.

Spec version: v2.2.0 · Run date: 2026-05-18 · Model: Claude Sonnet 4.6 (all builds) · Methodology: clean-room subagents per stage · Repo: github.com/itaki/botspeak

What changed in v2.2.0

Polarity inversion fixed. In v2.1.0, doc 05 marked DISABLE_AUTOUPDATER=1 with the !! "forbidden" symbol because the source language sounded cautionary. But that variable is an opt-out, not a prohibition. v2.2.0 adds a polarity verification step in the skill: substitute the literal word "forbidden" for each !! and verify the resulting statement holds.

Code-block parity enforced. In v2.1.0, doc 06's compression dropped 18 of its 20 Mermaid diagrams, YAML configs, and SQL snippets even though the skill said to preserve code blocks. v2.2.0 adds an explicit count check in the verify pass: count fenced blocks in source vs output and fail if the counts disagree. The re-run preserves 20 of 20.
